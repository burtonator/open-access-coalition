Firehose Costs

A firehose API provides a raw stream of updates to content published in (near) real time. This stream of content can be faster and cheaper for websites like Twitter or Status.net to serve (vs having a crawler index the content).

One criticism that websites often make is that hosting raw and realtime access to their content would be prohibitively expensive. As we whill show, this is simply not the case.

Microblogging

At the time of this writing, Twitter is indexing about 300M tweets per day. If we were to break this down into the costs to serve this API we would have:

Bandwidth costs

posts per day: 300M
bytes per month 1: 1.2TB
price per month 2: $189

1. 150 bytes for the max post length multipled by number of posts per day multipled by 30 days in a month.
2. Bytes per month multipled by $0.15 which is Amazon EC2's highest tier for banwidth. In practice a production application would be much less expensive.

Hardware costs

The above numbers are just for bandwidth.

Hardware is another important cost. Generally speaking, one server handle multiple API customers.

Mbps for content transfer per client:1: 4.1Mbps
price per month 2: $189

1. bytes per month divided by the number of seconds in a months multiplied by 8 (since there are 8 bites per byte).
